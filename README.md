# Баракат — PWA (готовое веб-приложение)

Это рабочее прогрессивное веб-приложение (PWA): ставится на экран телефона,
открывается как обычное приложение (без адресной строки браузера) и работает
офлайн благодаря service worker.

## Что уже работает по-настоящему
- Установка на главный экран (Android/iOS)
- Офлайн-режим — после первого открытия все экраны работают без сети
- Голосовой ввод и озвучка (Web Speech API — в Chrome на Android)
- Все 5 экранов: Главная, Диагностика, Голос, Календарь, Цены

## Что имитировано (для реального продукта нужно заменить)
- «AI-диагностика» — сейчас случайный выбор из готовой базы 10 болезней.
  Для настоящего распознавания по фото нужна модель (TensorFlow Lite/ONNX
  в браузере через TensorFlow.js), обученная на реальных фото растений.
- Цены — статичные демо-данные, не подключены к реальному источнику.

## Шаг 1. Разместить в интернете (нужен HTTPS)

PWA обязательно должен отдаваться по HTTPS (кроме localhost). Варианты,
бесплатные и без опыта в деплое:

**GitHub Pages**
1. Создать репозиторий, загрузить все файлы из этой папки в корень.
2. Settings → Pages → Source: ветка `main`, папка `/root`.
3. Через пару минут сайт будет на `https://<ваш-логин>.github.io/<репозиторий>/`.

**Netlify / Vercel** — ещё проще: зарегистрироваться, перетащить папку
в веб-интерфейс (drag-and-drop deploy), получить готовую ссылку.

Проверьте на телефоне: откройте ссылку в Chrome → должно появиться
предложение «Добавить на главный экран».

## Шаг 2. Публикация в RuStore

RuStore принимает не только APK, но и **веб-приложения (PWA)** — не нужно
собирать APK вручную. Проверьте актуальные требования и загрузите ссылку
на ваш задеплоенный PWA в личном кабинете разработчика:
- Документация и вход для разработчиков: https://www.rustore.ru/help/developers
- Понадобится: ИНН/самозанятость или юрлицо для регистрации разработчика,
  ссылка на политику конфиденциальности (нужно будет написать отдельно,
  так как приложение запрашивает камеру и микрофон), скриншоты экрана
  (можно сделать прямо из этого PWA на телефоне).

Если RuStore на момент публикации требует именно APK-обёртку — самый
быстрый путь превратить этот PWA в APK без написания кода на Kotlin:
инструменты **Bubblewrap** (Google, официальный) или **PWABuilder**
(pwabuilder.com) — оба берут манифест и генерируют подписанный APK
из готового PWA за несколько минут.

## Локальная проверка перед деплоем
Открыть папку в любом статическом сервере, например:
```
npx serve .
```
и зайти на `http://localhost:3000` (Service Worker для полной офлайн-работы
корректно активируется только по HTTPS или на localhost).
